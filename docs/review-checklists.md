# 表格与图片结果审查清单

## 执行前审查

- [ ] 已读取 GitHub 记忆，而不是只靠聊天历史。
- [ ] 当前任务使用的工作流文档已确认。
- [ ] 输入表格路径正确。
- [ ] 输出路径在店铺目录或 D 盘，不写爆 C 盘。
- [ ] 过程表命名包含 `过程` / `复检前` / `候选`，不会入库。

## T 首图审查

- [ ] 所有唯一 D 首轮 image2/APIMart。
- [ ] Seedream/即梦只处理 image2 失败或用户指定重做项。
- [ ] 每个 D 有 provider/model/source_png/prompt/status。
- [ ] 没有直接拿旧 Seedream/ComfyUI/阿里贴图填新表。
- [ ] 同一 L0xx 已轮换不同 source PNG。
- [ ] reject lock 未被绕过。
- [ ] 同 L0xx 的场景/色调/构图有明显差异。
- [ ] 产品放置场景符合真实用途。

## J 五格审查

- [ ] 每一行按 G/变体/SKU 货号匹配 SKU PNG。
- [ ] 黑白灰绿粉紫木色等颜色无错配。
- [ ] 五格图无黑色硬阴影。
- [ ] 产品尺寸比旧版更大，不太小。
- [ ] 不用 AI 生 J。

## T4/U/回填审查

- [ ] T1 是新通过首图。
- [ ] U = T1。
- [ ] T4 是尺寸图。
- [ ] 删除 T 后重新校验 T4。
- [ ] 同 D 多行 T1 一致。
- [ ] OSS URL 可访问。

## 入库审查

- [ ] 只有最终确认表入 D 查询库。
- [ ] 过程表不入库。
- [ ] 指纹复制能定位完整 D 变体行。
- [ ] 同 D 多最终表并存时不自动否定。

## 失败处理

- [ ] 用户反馈已写入 feedback/reject lock。
- [ ] 明确错误图不再回流。
- [ ] PNG 源素材错误时锁源，不只锁输出图。
- [ ] 修改后的规则已提交 GitHub。
