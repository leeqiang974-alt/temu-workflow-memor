# Excel 表格完整工作流

## 输入

- 店铺文件夹：`D:\Desktop\jit\{店铺名}`，例如 `DXXmall`。
- 源表或最终确认表：`.xlsx`。
- SKU PNG 根目录：按 `L0xx` 和变体存放。
- T 产品素材 PNG 库：象寄/新原图/用户筛选保留后的产品底片库。
- OSS 上传配置。

## 输出

- 过程表：文件名必须带 `过程_不入库` 或 `复检前`。
- 复检页：T/J 全局可视化审核，支持删除/恢复/导出反馈。
- 最终表：用户确认后才进入 D 查询/指纹库。
- 校验报告：JSON + HTML 摘要。

## 执行顺序

1. 读取 Excel，按精确 `产品货号(D)` 分组。
2. 标题重构。
3. 生成 T 首图候选。
4. 用户复检 T；失败项走 fallback。
5. 重做 J 五格预览图。
6. 校验 T 第四张必须是尺寸图。
7. U 列跟随 T 首图。
8. 压缩到平台要求，上传 OSS。
9. 回填 Excel。
10. 全局复检页面。
11. 用户确认后，最终表才允许入库。

## 入库规则

- 复检过程表、临时表、失败表、候选表禁止入 D 查询库。
- 只有最终确认并准备提交/已经提交的表格入库。
- 同一 D 出现在多个历史最终表是允许的，不能自动否掉。
- 指纹用于定位原始行，不用于否定后续表格中的同 D。

## 校验项

- 每个 D 的 T 首图一致：同 D 多行必须同 T1。
- U = T1。
- T4 是尺寸图。
- J 按行变体匹配，不按 D 盲用。
- T 和 J URL 可访问。
- 店小秘/外链旧图按规则替换或保留。
- X/AC 等辅助列按最新店铺规则校验。
