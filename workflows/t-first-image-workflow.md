# T 首图生成工作流

## 目标

为每个唯一 D 生成一张差异化、符合产品真实使用场景、产品结构尽量一致的 T 首图。

## 模型顺序

1. **首轮：image2/APIMart GPT-Image-2**
   - 对新表全部 D 全量生成。
   - 不能直接用旧 Seedream 库、ComfyUI 背景贴图、阿里单 SKU 图、all_sku_tfirst 结果跳过首轮。
2. **复检失败：Seedream/即梦 fallback**
   - 仅用于 image2 复检失败、结构错误、太相似、不够高级、场景不真实的 D。
   - 用户在复核页标记 `redo`、`不要`、结构错、场景没差异化、产品外观错的 D，必须先进入反馈锁，再作为 Seedream/即梦 fallback 或指定重做计划；不能把这批 image2 图写回最终表。
3. **极高风险结构：保守合成或固定 PNG 方案**
   - L042、L043、L078、L082、L088、L091 等高风险品类必须谨慎。

## 素材选择

优先级：

1. 用户筛选保留的新原图/象寄产品 cutout PNG。
2. retry_new_original 中未被拒绝的素材。
3. SKU variant PNG，仅当没有产品素材或用户明确要求时使用。

同一 L0xx 下：

- 必须轮换不同 PNG；这是全局规则，不只针对 L043。
- 不允许全 L0xx 只用一张 SKU 预览图。
- source_png/source_kind/source_id 必须写入结果记录。
- 每个 L0xx 在执行作图前必须先产出素材调配计划：列出每个 exact D 使用的 `source_png/source_kind/source_id`、场景 lane、色调 lane、构图/位置/比例差异。没有这个计划，不允许开跑。
- 同一 L0xx 必须同时满足两类差异化：`PNG 素材差异化` 和 `场景差异化`。不能只换场景不换 PNG，也不能只换 PNG 但画面场景模板化。
- 如果用户指出某个 L0xx 的 PNG 素材“太统一”，下一轮重构必须扩大可用素材池，优先切换不同 `kept_cutout` / `retry_new_original` / 安全产品 cutout，不得继续用同一外观源图套不同 D。
- 当前已发现：L042、L043 都存在 PNG 素材过于统一风险；这批已生成结果暂不返工，但下一次执行必须先做素材调配并过审。

## Prompt 关键规则

- 产品融合进真实使用场景，不是贴图。
- 场景必须符合产品属性和真实使用位置。
- 高价值感来自环境、空间、无品牌安全道具，不来自改变产品结构。
- 可替换内容只限松散可移除物：碗、杯、毛巾、文件、果蔬、花草等。
- 产品硬件必须冻结：框架、杆、孔、层数、篮、轮、把手、滑轨、支撑、颜色、轮廓。
- 色调必须轮换：冷白、蓝灰、绿色户外、深色高级、暖木、奶油马卡龙、黑白对比、中性阴天。

## 失败锁

以下反馈必须进入 reject lock：

- 产品结构改变。
- 缺杆、缺孔、孔位错误、顶部结构臆想。
- 场景不符合真实用途。
- 与同 L0xx 太相似。
- 用户明确说：不要、死刑、放弃、错、臆想。

## 当前已知高风险规则

- L042：围挡孔洞、固定片、钉子/桩不能重构。
- L043：折衣板孔洞、小孔、后凸起不能变；不能把折衣板变成普通垫板、盒子、托盘或比例离谱的大板。该组已反馈“PNG素材太统一”，后续重构必须切换不同源 PNG，至少轮换白/灰或不同角度/不同后凸起可见素材；如果素材池不足，先补素材或改用固定 PNG 合成，不要用同一张外观图硬生成多个 D。
- L047：只能一个拱门；可增加花/丝带但不能多造拱门。
- L058：桶、杆、篮、把手都容易被重构。
- L063：健身板不要人物使用图；偏健身房/训练室空场景。
- L078：用带黑色提手盖子的素材，严禁重构盘内结构。
- L082：橱柜下/隔层使用场景，不能臆想滑轨。
- L088：竖杆/中间杆位置不能错。
- L091：不要变角度，顶部结构/凹槽最容易错。

## 外部审查闸门

在项目开发、脚本修改、表格处理、图片结果放行前，必须把计划或结果交给 Claude Code + NVIDIA 审查提示词做一次独立判断。审查必须拦截：

- 未先读取 GitHub 记忆就开始执行。
- T 首图绕过 image2/APIMart 首轮，直接使用旧 Seedream/ComfyUI/阿里/all_sku_tfirst。
- image2 复检失败项未进入 feedback lock 就被写回或复用。
- 同一 L0xx 素材未轮换，或只有场景差异、没有 PNG 素材差异。
- 没有在作图前提交 source PNG 调配计划、场景 lane、色调 lane、构图差异计划。
- L042/L043 或任意被用户点名“PNG 太统一”的组继续使用过于统一的 PNG。
- 复核页导出反馈时没有读取当前输入框中文备注，导致用户填写的中文原因丢失。
- T4 尺寸图、U 跟随 T1、J 行级 SKU 变体匹配未校验。

## 作图执行前强制过审

后续每次真正开跑 T 首图、Seedream fallback、image2 redo、批量重构前，都必须先把执行计划交给 Claude Code + NVIDIA 审查。审查包必须包含：

- GitHub 记忆和当前 workflow 已读取的证据。
- redo / fallback 的 D 清单与用户中文反馈。
- 每个 L0xx 的 source PNG 调配表，证明 PNG 差异化。
- 每个 L0xx 的场景、色调、构图、产品比例计划，证明场景差异化。
- 高风险结构冻结规则和产品专项 prompt。
- 明确说明本批是否只是沿用已生成图；本批 199 image2 首轮已生成结果不因本条新规则自动返工，但下一次执行必须执行本规则。
